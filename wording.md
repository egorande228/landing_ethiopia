# 02 — Copy and Wording Profile

Purpose: define how landing-page wording should be structured, stored, localized, and separated by audience.

This file enforces visitor-facing language.

Both Home and Partnership pages are written for real visitors:

* Home page: players looking for games, sports, rewards, fast access, and support.
* Partnership page: potential partners, agents, promoters, cashiers, local operators, or workers interested in joining the opportunity.

The copy must not sound like it is explaining the website to the developer, reviewer, or coding agent.

This file controls:

* visitor-facing tone
* player vs partner/agent language separation
* terminology consistency
* content ownership
* CTA wording behavior
* FAQ wording behavior
* localization tone
* forbidden meta/dev-facing phrasing
* what copy belongs in structured content

This file does not control:

* section order
* typography sizes
* visual style
* colors
* images
* animation
* layout implementation

---

# 1. Content Ownership

All copy and data must come from structured content.

Do not put strings directly inside components.

Structured content owns:

* SEO title and description
* navigation labels
* hero copy
* section labels
* CTA labels
* card titles and bodies
* stats and metric labels
* footer copy
* contact methods
* FAQ questions and answers
* offer wording
* partnership route wording
* legal/commercial notes if needed

Components render supplied config only.

Components must not invent:

* market-specific copy
* business claims
* CTA labels
* role names
* offer values
* FAQ answers
* navigation wording

---

# 2. Main Copy Rule: Visitor-Facing Language

Every line of copy must sound like it is written for the visitor.

The visitor is not the developer.

The visitor does not care that:

* the section is clean
* the design was improved
* the layout explains something
* the card presents information
* the page contains sections
* the system uses a structure

The visitor cares about:

* what they can do
* what they can get
* how fast they can start
* why they should trust it
* which route fits them
* how to contact or join

Do not write copy that describes the page itself.

## Forbidden meta wording

Avoid phrases like:

this section shows
this page explains
the user can see
the design includes
the layout provides
the cards display
we included
the platform offers a section
here you will find
this block highlights
this area presents
this page was built to
the content below explains

## Correct direction

Instead of describing the section, speak directly to the visitor.

Bad:

This section shows the benefits available for players.

Good:

ابدأ بسرعة، اختر لعبتك، واستفد من العروض المتاحة لك.

Bad:

The partnership page explains the available work routes.

Good:

اختر الطريق المناسب لك: شريك يروّج لجمهوره، أو وكيل يدير التعاملات محلياً.

Bad:

The final CTA allows users to contact us.

Good:

تواصل الآن، وسنساعدك على اختيار المسار المناسب والبدء بخطوات واضحة.

# 3. Audience Separation

Both Home and Partnership are visitor-facing.

The difference is the visitor’s intent.

## 3.1 Home Page Audience

Home speaks to players.

A Home visitor is asking:

* Can I start quickly?
* What games are available?
* Is there sports betting?
* Are there casino games?
* Are there gifts, free spins, free bets, or bonuses?
* Is support easy?
* Is it mobile-friendly?
* What do I do next?

Home should sound like:

* fast access
* sports and games
* rewards
* free bets
* free spins
* gifts
* play flow
* easy registration
* support
* mobile experience

Home must not sound like:

* partner recruitment
* affiliate management
* business operations
* internal documentation
* a developer brief
* a design explanation

## 3.2 Partnership Page Audience

Partnership speaks to people interested in joining the work.

A Partnership visitor may be:

* a partner
* an agent
* an affiliate partner
* a local promoter
* a cashier
* a community owner
* a content creator
* a person with local payment access
* someone who can bring players
* someone looking for an opportunity

A Partnership visitor is asking:

* Can I work with this company?
* Do I fit as a partner or agent?
* What exactly would I do?
* How can I earn?
* What tools/support will I get?
* Who helps me?
* How do I start?
* How do I contact someone?

Partnership should sound like:

* opportunity
* work route
* earning path
* local role
* manager support
* tools
* tracking
* promo code
* local transactions
* operational support
* clear next step

Partnership must not sound like:

* player entertainment copy
* corporate business-deck language
* generic startup marketing
* internal notes
* a section explanation
* an instruction to the developer

# 5. Clean Language Rule

Avoid vague “more” wording.

The copy should feel clean because it is specific, not because it says “clean” or “more”.

Avoid weak phrases like:

* more better
* more premium
* more clean
* more exciting
* more professional
* more opportunity
* more support
* more value
* more tools
* more rewards
* more trusted

Use concrete phrasing instead.

Bad:

Get more rewards and more support.

Good:

احصل على مكافآت واضحة، دعم سريع، وتجربة لعب سهلة من أول يوم.

Bad:

A more professional way to work with us.

Good:

مسار واضح للعمل، أدوات جاهزة، ومدير يساعدك في المتابعة.

Bad:

More tools for partners.

Good:

أدوات تسويق، كود ترويجي، تتبع للأداء، ودعم من مدير مختص.

Bad:

More benefits for agents.

Good:

طرق دفع مناسبة، متابعة مباشرة، ودور واضح في خدمة اللاعبين محلياً.

# 6. Home Page Copy Rules

Home copy should answer:

1. Why should a player care immediately?
2. What can the player explore?
3. What sports action is available?
4. What reward or offer exists?
5. What should the player do next?

Recommended tone:

* direct
* energetic
* premium
* simple
* conversion-oriented
* not exaggerated
* visitor-facing

Home page copy should focus on:

* fast sign-up
* sports
* casino
* free bets
* free spins
* gifts
* first deposit bonus if configured
* mobile access
* provider quality
* support
* easy start

Avoid:

* business recruitment copy
* “this section shows”
* dev-facing wording
* repeated claims
* fake guarantees
* explaining the design or layout
* corporate partnership language

Good Home phrasing direction:

ابدأ بسرعة، تابع المباريات، جرّب ألعاب الكازينو، واستفد من العروض المتاحة لك.

كل ما تحتاجه للعب أسرع: تسجيل بسيط، ألعاب قوية، عروض واضحة، ودعم قريب منك.

مباريات مباشرة، ألعاب كازينو، مكافآت، وتجربة مصممة للهاتف من أول خطوة.

# 7. Partnership Page Copy Rules

Partnership copy should answer:

1. What is the opportunity?
2. Who is it for?
3. Which route fits me?
4. What support/tools do I get?
5. How are deals generally handled?
6. How do I start?

Recommended tone:

* professional
* clear
* confident
* practical
* sales-focused without sounding fake
* opportunity-facing, not corporate
* visitor-facing

Partnership copy should speak to someone asking:

* Can I work with this?
* Do I fit as a partner or agent?
* What will I receive?
* Who helps me?
* How do I contact them?
* Is this serious enough to start?

Do not repeat the same business promise in hero, benefits, and final CTA without adding new information.

Do not write as if explaining the page structure.

Bad:

This page explains the partner and agent models.

Good:

اختر المسار الأقرب لك: شريك يملك جمهوراً ويروّج بكود خاص، أو وكيل يدير تعاملات اللاعبين محلياً.

Bad:

The final CTA allows users to contact us.

Good:

تواصل الآن، وسنساعدك على اختيار المسار المناسب والبدء بخطوات واضحة.

Good Partnership phrasing direction:

حوّل جمهورك أو شبكتك المحلية إلى مسار عمل واضح، مع أدوات جاهزة ودعم من مدير مختص.

سواء كنت تملك جمهوراً أو تستطيع إدارة تعاملات اللاعبين محلياً، هناك مسار واضح للبدء.

ابدأ بكود ترويجي، أدوات تسويق، متابعة أداء، ودعم يساعدك على النمو خطوة بخطوة.

# 8. Partner vs Agent Copy Separation

Use separate route cards or sections when both are relevant.

## 8.1 Partner

A partner route usually explains:

* audience/community promotion
* promo code or traffic-driving model
* affiliate-style promotion
* manager support
* campaign materials
* performance visibility

Partner copy should speak to:

* influencers
* community owners
* content creators
* local promoters
* people with traffic or audience reach

Partner visitor question:

* I have an audience or traffic. How can I promote and earn?

Good direction:

روّج لجمهورك بكود خاص، وتابع الأداء مع مدير يساعدك في تحسين النتائج.

إذا كان لديك جمهور، مجتمع، قناة، أو حركة زيارات، يمكنك تحويلها إلى شراكة واضحة بأدوات تتبع ودعم تسويقي.

## 8.2 Agent

An agent route usually explains:

* local transaction support
* cashier-style role
* connection between players and platform
* local payment methods
* operational trust
* city/country presence

Agent copy should speak to:

* people who can handle local transactions
* local cashiers
* people with payment access
* people who can support players in a city or country

Agent visitor question:

* I can handle transactions locally. How can I work?

Good direction:

كن نقطة وصل محلية للاعبين، وساعدهم في الإيداع والسحب عبر الطرق المناسبة في منطقتك.

إذا كنت تستطيع إدارة تعاملات اللاعبين محلياً، يمكنك العمل كوكيل يساعد في الإيداع والسحب والدعم المباشر.

Keep the two routes distinct.

Do not merge them into one vague “work with us” block unless the page is intentionally minimal.

# 9. Financial and Offer Wording Rules

Specific numbers belong in market content or market facts files, not in components.

When using financial or offer values:

* phrase carefully
* avoid fake guarantees
* avoid guaranteed-income language
* avoid implying every visitor receives the same deal
* explain that terms depend on agreement, volume, role, and manager approval where relevant

## 9.1 Player offers

Examples of player offers may include:

* gifts
* free spins
* free bets
* first deposit bonus
* provider quality
* fast signup
* support

Use direct offer language.

Good:

استفد من عروض الترحيب، الرهانات المجانية، واللفّات المجانية حسب العروض المتاحة.

Bad:

Guaranteed huge winnings for every player.

## 9.2 Partner terms

Partner deal language may include:

* revenue share
* CPA
* hybrid deal
* promo code
* manager support

Use flexible language.

Good:

تختلف شروط الشراكة حسب نوع الشريك، حجم الجمهور، وطريقة الاتفاق مع مدير الشراكات.

Bad:

Every partner gets 40% revenue share.

## 9.3 Agent terms

Agent deal language may include:

* deposit handling
* withdrawal handling
* local payment methods
* commission ranges if configured

Use careful language.

Good:

تُحدد النسب وطريقة العمل حسب المدينة، حجم التعاملات، وطرق الدفع المتاحة.

Bad:

Every agent earns a fixed percentage forever.

# 11. FAQ Copy Rules

FAQ must be separated by audience:

1. Player FAQ
2. Partnership FAQ

FAQ answers should be:

* short
* practical
* direct
* not promotional essays
* not internal documentation
* not explanations of the page layout

Player FAQ topics:

* how to start
* available games
* sports/casino availability
* promos/offers
* mobile access
* support/help

Partnership FAQ topics:

* who can apply
* partner vs agent difference
* tools and support
* tracking/performance
* deal terms at a high level

FAQ should not sound like the page is explaining itself.

Bad:

This FAQ helps users understand the page.

Good:

أسئلة سريعة قبل أن تبدأ كلاعب أو تتواصل كشريك.

يمكنك البدء بخطوات بسيطة، ثم اختيار الرياضة أو اللعبة المناسبة لك.

Good Partnership FAQ answer style:

يمكنك التواصل مع المدير لاختيار المسار الأنسب: شريك ترويجي أو وكيل محلي.

# 12. Localization Rules

Localized copy should remain natural in the active language.

Rules:

* Arabic copy should not sound like literal English translation.
* English fallback should remain simple and clean.
* Do not mix Arabic and English inside the same sentence unless it is a brand/product term.
* Numbers, percentages, phone numbers, currency values, scores, timers, and range values remain LTR.
* Locale switching should preserve page intent and anchor when practical.

Arabic should feel like it was written for Arabic-speaking visitors, not translated mechanically.

Avoid stiff Arabic phrases like:

تقوم هذه الصفحة بتقديم
يوفر هذا القسم
يمكن للمستخدم مشاهدة
تهدف هذه الصفحة إلى
يعرض هذا التصميم

Prefer direct phrasing:

ابدأ بسرعة
اختر مسارك
تواصل مع المدير
تابع الأداء
استفد من العروض
اكتشف الألعاب
تابع المباريات

# 13. English Fallback Rules

English fallback should be:

* simple
* direct
* not overly corporate
* not over-explained
* not written like internal documentation

Bad:

This section explains the available player benefits.

Good:

Start fast, explore the games, and claim available rewards.

Bad:

The partnership model is presented through two cards.

Good:

Choose your route: promote with a private code, or support local player transactions.

# 14. Writing Quality Rules

Good copy is:

* visitor-facing
* specific enough to be useful
* short enough to scan
* aligned with section purpose
* not repeated from previous section
* not written like a dev note
* not written like a template explanation
* not written like it is convincing the developer that the job was done

The copy should not say or imply:

* “we made this clean”
* “this layout is better”
* “the section is improved”
* “this page is designed to”
* “the content now shows”
* “the cards help explain”
* “this makes the experience more premium”
* “the page now feels more local”

The page should simply speak to the visitor.

Avoid:

* “this section shows”
* “the user can see”
* “the design includes”
* “placeholder”
* “lorem ipsum”
* vague hype
* unrealistic guarantees
* internal task wording
* review-response language
* self-congratulatory language

Bad:

A cleaner layout that helps partners understand the models.

Good:

مساران واضحان للعمل: شريك يروّج بكود خاص، أو وكيل يدير التعاملات محلياً.

---

# 15. Repetition Control

Do not repeat the same promise across multiple sections.

Each section must add new information.

Bad flow:

* Hero: Work with us and grow.
* Benefits: Work with us and grow.
* Paths: Work with us and grow.
* Final CTA: Work with us and grow.

Good flow:

* Hero: introduces the opportunity.
* Benefits: explains why it is supported.
* Paths: separates Partner and Agent.
* Tools: explains what the visitor receives.
* Final CTA: gives the contact action.

Rule:
If a sentence could appear in any section, it is probably too generic.

# 17. Business Specifics

This file intentionally does not define market-specific financial percentages, deposit offers, or exact bonus claims.

Those belong in:

* market content files
* market facts files
* campaign configuration
* legal/commercial copy files if needed

When using specific claims:

* phrase carefully
* do not imply guarantees unless confirmed
* place values in structured content
* keep legal/commercial flexibility where needed

For partnership financial terms:

* use careful phrasing
* explain that exact terms depend on agreement, volume, role, and manager approval
* avoid guaranteed-income language

For player offers:

* avoid unrealistic claims
* avoid fake urgency
* avoid guarantees unless confirmed
* use clean direct offer language
